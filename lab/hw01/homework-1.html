Answer to Question

Q: Explain in which cases (about the different parameters for decentralized learning) the two are equivalent.

A: FedSGD: Clients compute gradients after one mini-batch and send them to the server, which averages and applies the update.

FedAvg: Clients train locally for multiple epochs, send updated model weights to the server, which then averages and updates the global model.

FedAvg and FedSGD give the same results when E=1 because each client does just one update before sending it, making FedAvg work like FedSGD. If C is high, both methods get updates from almost the same number of clients, so the results match. Basically, when clients dont train much locally and enough of them join each round, FedAvg and FedSGD are the same.

import pandas as pd

def run_experiment(n, c, algorithm, seed=10):
    client_subsets = split(n, True, seed)
    if algorithm == "FedSGD":
        server = FedSgdGradientServer(lr, client_subsets, c, seed)
    else:  # FedAvg
        server = FedAvgServer(lr, b, client_subsets, c, e, seed)
    
    result = server.run(nr_rounds=nr_rounds)
    # result = server.run(nr_rounds=2)
    
    message_count = result.message_count
    test_accuracy = result.test_accuracy
        
    return sum(message_count), test_accuracy
